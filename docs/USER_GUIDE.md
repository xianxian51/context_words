# 语境单词本使用指南

## 每日学习

默认情况下，App 每天第一次打开会自动检查并准备今日计划。配置 DeepSeek API Key 且开启自动阅读后，App 会为当前批次补齐阅读预热和语境强化短文；已有计划或阅读会直接复用。

首页“再来一组”会保留旧批次并追加新单词。网络或 DeepSeek 请求失败不会删除已经生成的计划，可稍后重试缺失阅读。

## DeepSeek API Key

App 不内置 API Key。请在“设置”页面填写自己的 DeepSeek API Key。Key 只保存在本机，不会进入学习数据备份，也不应提交到 GitHub。

设置页可选择 DeepSeek 模型：默认 `deepseek-v4-pro` 质量更高；`deepseek-v4-flash` 响应更快、更省费用。释义补全、点词查询、阅读、翻译、易混词辨析和英语助手都会使用同一模型设置。

## 英语助手

首页或设置页可进入“英语助手”。输入问题并点击发送后才会消耗 DeepSeek token；未发送的文字不会上传。回复支持 Markdown、复制和失败重试，当前聊天最多携带最近 10 轮上下文。

## 词库与复习

- 在词库页可按英文、中文释义、英文释义或词性搜索。
- 今日单词页用于查看当天目标词；复习页用于标记已掌握、模糊或不认识。
- 单词可星标进入重点词册，也可加入自定义单词本或易混词组。
- 阅读、英文释义和例句中的英文词可点击查询；本地没有时，只有用户确认后才调用 DeepSeek。

## 阅读、翻译与生成短文

- 今日两轮阅读、自定义单词本短文和易混词组对比短文支持目标词高亮与点词查询。
- 全文翻译首次请求或重新翻译会消耗 DeepSeek token；已保存翻译可离线查看。
- 自定义单词本和易混词组会限制单次发送的词数，避免不必要的 token 消耗。
- 易混词 AI 辨析采用适合手机阅读的精简 Markdown，不再生成小测验。

## 自动准备与学习入口

- “设置 → 每日首次打开自动准备学习内容”默认开启；当天已有计划时不会重复生成。
- “自动生成阅读”默认开启；缺少 API Key 时只准备单词，不调用 DeepSeek。
- “再来一组”保留旧批次，追加新单词，并只为新批次补齐缺失的两篇阅读。
- 首页只保留开始学习、今日单词、再来一组和英语助手等常用入口；导入、重建和手动生成阅读位于“更多 / 管理 / 高级操作”。

## 搜索与显示更多

词库搜索会查询本地 Repository，支持英文、中文释义、英文释义和词性，并按页加载结果。易混词组的候选搜索可反复点击“显示更多”，已选择项目不会因加载下一页而丢失。

## 检查更新

“设置 → 检查更新”可手动查询 GitHub Releases；“启动时检查更新”默认开启。有新版本时 App 只显示 Release 摘要并引导打开下载页面，不会静默下载或安装。网络失败不会影响日常学习。

## TTS 发音

在“设置 → 发音设置”中可选择美式发音 `en-US`、英式发音 `en-GB` 或跟随系统。默认使用美式发音，“测试发音 academic”可用于确认当前效果。

### 发音不一致怎么办

1. 本 App 使用手机系统 TTS，不同手机的 TTS 引擎和语音包不同，声音可能不同。
2. 推荐在设置页选择“美式发音 en-US”。
3. 如果“当前使用”只显示 `en`，说明手机缺少具体的美式或英式语音包，该发音只是临时降级。
4. 请打开安卓“设置 → 语言和输入法 → 文字转语音输出”，下载或选择 Google 文字转语音 / 英语（美国）`en-US` 语音包。
5. 如果发音无声，同时检查媒体音量，然后点击“刷新 TTS 状态”和“测试发音 academic”。
6. 如果需要在所有设备上完全统一声音，后续需要接入统一音频文件，而不是系统 TTS。

## 数据备份与恢复

在“设置 → 数据管理”中：

1. 点击“导出学习数据”保存 JSON 备份。
2. 点击“导入学习数据”选择备份。
3. 导入采用合并策略，不会先清空当前数据库。

备份不包含 DeepSeek API Key。

## 从旧测试包名迁移到正式包名

- 旧测试包名：`com.example.context_words`
- 正式包名：`io.github.xianxian51.contextwords`

Android 不会在两个不同 applicationId 之间自动迁移 SQLite、设置或学习记录。迁移步骤：

1. 保留旧测试版，在旧版“设置 → 数据管理”中导出学习数据。
2. 安装正式包名版本；它会作为一个新的 App 出现在手机上。
3. 打开正式版，在“设置 → 数据管理”中导入刚才的备份。
4. 核对星标、单词本、易混词组和历史记录后，再决定是否卸载旧测试版。

不要先卸载旧测试版，否则 Android 会删除其本地数据。

## 后续更新不丢数据

正式包名确定后，只要后续 APK 继续使用 `io.github.xianxian51.contextwords`、使用相同签名证书并提高 versionCode，覆盖安装通常会保留本地数据。卸载 App 会删除系统管理的本地数据，更新前仍建议导出备份。

## GitHub 测试版 APK

GitHub Release 中的 `app-debug.apk` 仅用于测试体验，不适合正式生产发布。Debug 签名可能随构建机器变化，无法保证长期覆盖升级；正式发布仍需要稳定的 Release keystore。
