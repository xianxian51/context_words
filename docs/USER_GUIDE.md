# 语境单词本使用指南

## 每日学习

默认情况下，App 每天第一次打开会自动检查并准备今日计划。配置 DeepSeek API Key 且开启自动阅读后，App 会为当前批次补齐阅读预热和语境强化短文；已有计划或阅读会直接复用。

首页“再来一组”会保留旧批次并追加新单词。网络或 DeepSeek 请求失败不会删除已经生成的计划，可稍后重试缺失阅读。

## DeepSeek API Key

App 不内置 API Key。请在“设置”页面填写自己的 DeepSeek API Key。Key 只保存在本机，不会进入学习数据备份，也不应提交到 GitHub。

## 词库与复习

- 在词库页可按英文、中文释义、英文释义或词性搜索。
- 今日单词页用于查看当天目标词；复习页用于标记已掌握、模糊或不认识。
- 单词可星标进入重点词册，也可加入自定义单词本或易混词组。
- 阅读、英文释义和例句中的英文词可点击查询；本地没有时，只有用户确认后才调用 DeepSeek。

## 阅读、翻译与生成短文

- 今日两轮阅读、自定义单词本短文和易混词组对比短文支持目标词高亮与点词查询。
- 全文翻译首次请求或重新翻译会消耗 DeepSeek token；已保存翻译可离线查看。
- 自定义单词本和易混词组会限制单次发送的词数，避免不必要的 token 消耗。

## TTS 发音

如果发音无声，请检查手机媒体音量和系统文字转语音引擎，并在“设置”页面点击“测试发音”。Android 通常可在“语言和输入法 → 文字转语音输出”中选择可用引擎。

## 数据备份与恢复

在“设置 → 数据管理”中：

1. 点击“导出学习数据”保存 JSON 备份。
2. 点击“导入学习数据”选择备份。
3. 导入采用合并策略，不会先清空当前数据库。

备份不包含 DeepSeek API Key。

## 从旧测试包名迁移到正式包名

- 旧测试包名：`com.example.context_words`
- 正式包名：`io.github.xianxian51.contextwords`

Android 不会在两个不同 applicationId 之间自动迁移 SQLite、设置或学习记录。迁移步骤：

1. 保留旧测试版，在旧版“设置 → 数据管理”中导出学习数据。
2. 安装正式包名版本；它会作为一个新的 App 出现在手机上。
3. 打开正式版，在“设置 → 数据管理”中导入刚才的备份。
4. 核对星标、单词本、易混词组和历史记录后，再决定是否卸载旧测试版。

不要先卸载旧测试版，否则 Android 会删除其本地数据。

## 后续更新不丢数据

正式包名确定后，只要后续 APK 继续使用 `io.github.xianxian51.contextwords`、使用相同签名证书并提高 versionCode，覆盖安装通常会保留本地数据。卸载 App 会删除系统管理的本地数据，更新前仍建议导出备份。

## GitHub 测试版 APK

GitHub Release 中的 `app-debug.apk` 仅用于测试体验，不适合正式生产发布。Debug 签名可能随构建机器变化，无法保证长期覆盖升级；正式发布仍需要稳定的 Release keystore。
